# Agentic Operations

## Overview

Unboxd Cloud introduces an operational layer where enterprise services can be paired with domain-specific operational agents.

The objective is to improve operational consistency, repeatability, visibility, and lifecycle automation across infrastructure and application services.

## Operational Scope

Operational agents can assist with:

- Provisioning
- Configuration
- Monitoring
- Service remediation
- Scaling operations
- Lifecycle management
- Documentation workflows
- Operational visibility

## Service Domains

Examples of operational domains:

| Domain | Example |
|---|---|
| Infrastructure | OpenStack operations |
| Edge Operations | MicroCloud orchestration |
| Identity | Keycloak and Logto workflows |
| APIs | APISIX management |
| Observability | SigNoz and GlitchTip operations |
| Publishing | WordPress and Ghost management |
| Commerce | WooCommerce operations |

## Governance Model

Operational agents are positioned as enterprise operational support systems.

They extend infrastructure and operations teams rather than replacing enterprise governance and administrative controls.

## Platform Direction

The long-term direction of the operational layer includes:

- Service-domain orchestration
- Agent registries
- Operational memory
- Workflow automation
- Multi-agent operational coordination
- Infrastructure-aware execution
